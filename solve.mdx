---
title: "Solve reCAPTCHA"
description: "Solve a reCAPTCHA v3 challenge and get a valid token."
api: "POST /api/solve"
---

Submit a reCAPTCHA v3 challenge and receive a valid token ready to use in your form submission.

## Request

<ParamField header="X-API-Key" type="string" required>
  Your API key.
</ParamField>

<ParamField body="site_key" type="string" required>
  The reCAPTCHA site key (starts with `6L`, 40 characters).
</ParamField>

<ParamField body="page_url" type="string" required>
  Full URL of the page with reCAPTCHA.
</ParamField>

<ParamField body="action" type="string">
  reCAPTCHA action string (default: `submit`).
</ParamField>

<ParamField body="proxy" type="string">
  Proxy address (e.g., `user:pass@host:port`).
</ParamField>

<ParamField body="proxy_type" type="string">
  Proxy protocol: `http`, `https`, `socks5`.
</ParamField>

## Example

<CodeGroup>

```python Python
import httpx

r = httpx.post(
    "https://api.bustedcap.com/api/solve",
    json={
        "site_key": "6LfmULUpAAAAAC_HEF5R4rz2blWdiRSDfq0XZ2xp",
        "page_url": "https://example.com/donate",
        "action": "donate",
    },
    headers={"X-API-Key": "sk_live_your_key_here"},
    timeout=60,
)

data = r.json()
print(data)
```

```bash cURL
curl -X POST https://api.bustedcap.com/api/solve \
  -H "Content-Type: application/json" \
  -H "X-API-Key: sk_live_your_key_here" \
  -d '{"site_key": "6LfmULUpAAAAAC_HEF5R4rz2blWdiRSDfq0XZ2xp", "page_url": "https://example.com/donate", "action": "donate"}'
```

</CodeGroup>

## Response

<ResponseField name="status" type="string">
  `"success"` when the solve completed.
</ResponseField>

<ResponseField name="token" type="string">
  The reCAPTCHA v3 token to use in your form submission.
</ResponseField>

<ResponseField name="time" type="float">
  Solve time in seconds.
</ResponseField>

<ResponseField name="cached" type="boolean">
  Whether a cached result was returned.
</ResponseField>

<ResponseField name="remaining" type="integer or null">
  Remaining requests on your key (`null` if unlimited).
</ResponseField>

## Errors

<AccordionGroup>

<Accordion title="400 -- Invalid site key format">
```json
{
  "detail": "Invalid site_key format"
}
```
The `site_key` must start with `6L` and be exactly 40 characters.
</Accordion>

<Accordion title="401 -- Invalid API key">
```json
{
  "detail": "Invalid API key"
}
```
The `X-API-Key` header is missing or does not match any active key.
</Accordion>

<Accordion title="403 -- Request limit reached">
```json
{
  "detail": "Request limit reached",
  "message": "You have used all 100 requests. Key removed. Contact admin for a new key."
}
```
Your key has exhausted its request quota.
</Accordion>

<Accordion title="429 -- Rate limit exceeded">
```json
{
  "detail": "Rate limit exceeded",
  "retry_after": 12.5
}
```
Too many requests in a short period; wait `retry_after` seconds before retrying.
</Accordion>

<Accordion title="503 -- Server busy">
```json
{
  "status": "error",
  "message": "Server busy, try again later"
}
```
All solver slots are occupied; retry after a brief delay.
</Accordion>

<Accordion title="500 -- Solve failed">
```json
{
  "status": "error",
  "message": "Failed to obtain token",
  "time": 15.2
}
```
The solver could not produce a valid token within the time limit.
</Accordion>

</AccordionGroup>

## Notes

<Info>
  - Tokens are **single-use** -- each solve returns a fresh token.
  - Recommended timeout: **60 seconds**. First solves for a new site may take up to 15s; subsequent solves are faster.
  - The `action` parameter must match what the target site expects. Check the site's JavaScript for `grecaptcha.execute(key, {action: "..."})` to find the correct action string.
</Info>
